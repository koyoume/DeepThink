# 생각 모음 — UI 설계 문서

> 개인 생각/아이디어 정리용 모바일 앱(안드로이드 기준)의 UI 프로토타입 설계.
> 본 문서는 현재까지 구현된 클릭형 프로토타입(`thoughts-prototype.html`)의 설계 내용을 정리한 것이다.

작성/갱신: 2026-06-30 · 상태: 프로토타입 v0.4

---

## 1. 개요

핵심 단위는 **주제(Topic)** 다. 각 주제는 하나의 카테고리에 속하며, 제목과 (선택적) 관련 자료, 그리고 여러 줄의 **생각(Thought)** 으로 구성된다. 생각은 한 줄 단위로 작성하며, 줄마다 **체크리스트** 또는 **코멘트** 타입을 가지고, 레벨(들여쓰기)로 상·하위 관계를 표현한다.

화면은 두 개로 구성된다.

- **대시보드**: 선택된 카테고리의 주제들을 2열 카드로 한눈에 표시
- **상세 화면**: 한 주제의 전체 내용을 보고 편집

---

## 2. 정보 구조 / 데이터 모델

```
Topic {
  cat:       string        // 카테고리 (단일)
  title:     string        // 주제 제목
  materials: Material[]     // 관련 자료 (선택, 0개 이상)
  thoughts:  Thought[]      // 생각 줄 목록 (순서 있음)
}

Material {
  kind:  'link' | 'doc'     // 자료 유형
  title: string
  sub:   string             // 출처/도메인 표기
}

Thought {
  id:    string
  type:  'check' | 'comment'   // 줄 아이콘 유형
  level: number                // 들여쓰기 레벨 (0 = 최상위)
  text:  string
  done:  boolean               // check 타입에서만 의미, 완료 여부
}
```

- 카테고리: `제품 기획`, `독서`, `투자`, `학습`, `일상`, `사이드 프로젝트` (프로토타입 샘플)
- `thoughts`는 평면 배열이며 트리 구조는 `level` 값으로 표현한다. 하위 줄은 바로 위 상위 줄의 `level + 1`을 넘을 수 없다.

## 3. 디자인 토큰

### 색상
| 용도 | 값 |
|---|---|
| 배경 | `#FBFAF7` |
| 카드/표면 | `#FFFFFF` |
| 본문 텍스트 | `#1F2421` |
| 보조 텍스트 | `#8A8F87` / `#6E726A` |
| 구분선 | `#ECEAE3` |
| 들여쓰기 가이드선 | `#E3E1D9` |
| 강조(주요 액션·체크) | `#2D6A5A` (pine) |
| 코멘트 | `#C9803A` (amber) |

### 카테고리 색 점
제품 기획 `#2D6A5A` · 독서 `#B0793C` · 투자 `#8E6F4E` · 학습 `#6E6A9E` · 일상 `#7A8A6E` · 사이드 프로젝트 `#5A7693`. 모두 저채도/어스톤으로 통일해 노이즈를 줄인다.

### 타이포그래피
- 본문: 산세리프 (Inter / Roboto / system)
- 상세 화면 주제 제목: 세리프 (Iowan Old Style / Georgia) — 사색적 인상
- 카테고리 라벨은 보조 회색으로 통일하고, 색은 점(dot)에만 부여

### 아이콘 (두 가지 줄 유형)
- **체크리스트**: 둥근 사각 체크박스. 완료 시 pine 채움 + 흰 체크 + 텍스트 취소선/회색 처리
- **코멘트**: 둥근 말풍선(amber 외곽 + 연한 amber 배경)
- 모양(사각 vs 말풍선)과 색(pine vs amber)으로 한눈에 구분

## 4. 대시보드 화면

### 구성 (위→아래)
1. 상태바
2. 헤더: 좌측 화면 제목(`생각 모음`), 우측 **검색** 아이콘 + **뷰 옵션** 버튼
3. 카테고리 필터 칩 (가로 스크롤)
4. 캡션 줄: `주제 N개` / `미리보기 · 최대 N줄`
5. 주제 카드 2열 그리드
6. 우하단 주제 추가 FAB

### 카테고리 칩 (단일 선택, 필수)
- "전체" 없음. **항상 하나의 카테고리가 선택**된 상태이며, 그리드는 해당 카테고리의 주제만 표시
- 기본 선택: 첫 카테고리(`제품 기획`)
- 선택 칩은 pine 채움, 나머지는 외곽선 칩

### 주제 카드
- 카드 안에는 **카테고리 정보를 표시하지 않는다** (상단에서 이미 카테고리를 선택했으므로 불필요)
- **제목**: 한 줄 출력. 폭에 맞춰 폰트 자동 축소(시작 18px → **가독 최소 11px**). 최소 크기에서도 넘치면 말줄임(…)
- **최근 생각 미리보기**: 제목 아래에 가장 최근 입력한 생각 최대 N줄(뷰 옵션 값). 줄마다 미니 아이콘 + 텍스트(완료는 취소선)
- 미리보기가 없는 주제는 제목만 있는 짧은 카드 → 가변 높이(메이슨리) 레이아웃
- 카드 탭 → 해당 주제 상세 화면으로 전환

### 뷰 옵션 (상단 우측 노출)
- 버튼을 누를 때마다 미리보기 줄 수가 `3 → 2 → 1 → 끔 → 3` 순환
- 카드 미리보기와 캡션이 즉시 반영 (※ 본래 줄 수 기본값은 설정 화면에서 지정하는 것으로 설계, 대시보드의 버튼은 빠른 전환)

## 5. 상세 화면

### 구성 (위→아래)
1. 상태바
2. 앱바: 좌측 뒤로가기(‹), 우측 더보기(⋯)
3. 카테고리 칩 (해당 주제의 카테고리, 색 자동)
4. **주제 제목** (고정 높이 박스)
5. 관련 자료 (있을 때만)
6. 생각 리스트
7. 하단 입력바 (맨 아래에 추가)
8. (길게 누르기) 줄 액션 시트

### 제목 박스 (고정 높이)
- 세로 높이 **64px 고정** → 제목 길이와 무관하게 아래 레이아웃 위치 일정
- 짧은 제목: 큰 폰트(27px) **한 줄**, 박스 내 세로 중앙 정렬
- 긴 제목: 폰트를 줄여 **두 줄**(최대 22px → 최소 15px)로 박스에 맞춤
- 최소 폰트에서도 넘치면 2줄 말줄임(…)

### 관련 자료 (선택)
- 라벨에 "선택" 표기. 링크/문서 카드를 리스트로 표시(아이콘 + 제목 + 출처 + › )

### 생각 리스트
- 줄 = 좌측 아이콘(체크/코멘트) + 텍스트
- 레벨당 들여쓰기 22px, 들여쓴 줄은 좌측 가이드선 표시
- 완료된 체크 줄은 취소선 + 회색
- 생각이 없으면 빈 상태 안내 + "+ 한 줄 추가" 제공

## 6. 편집 인터랙션 명세 (상세 화면)

| 동작 | 조작 | 결과 |
|---|---|---|
| 줄 편집 | 줄 텍스트 탭 | 인라인 편집 진입(커서) |
| 줄 추가 / 중간 삽입 | 편집 중 **Enter** | 바로 아래에 **같은 레벨** 새 줄 생성, 타입 상속, 커서 이동 |
| 입력 종료 | 빈 줄에서 **Enter** | 빈 줄 제거, 이전 줄로 이동 / 종료 |
| 줄 삭제 | 빈 줄 맨앞에서 **Backspace** | 줄 삭제, 이전 줄로 이동 |
| 들여쓰기(하위) | 줄을 **오른쪽으로 스와이프** | 레벨 +1 (위 줄의 레벨 +1 까지만, 최대 4단계). 하위 줄 함께 이동 |
| 내어쓰기(상위) | 줄을 **왼쪽으로 스와이프** | 레벨 −1 (최소 0). 하위 줄 함께 이동 |
| 완료 토글 | 체크 아이콘 탭 | `done` 토글 (check 타입만) |
| 줄 메뉴 | 줄 **길게 누르기**(≈480ms) | 액션 시트: 체크↔코멘트 전환 / 들여쓰기 / 내어쓰기 / 삭제 |
| 맨 아래 추가 | 하단 입력바 탭 | 레벨 0, 현재 선택 타입으로 새 줄 추가 후 편집 |
| 새 줄 기본 타입 | 입력바 좌측 토글(체크/코멘트) | 이후 추가되는 줄의 기본 타입 결정 |

규칙 메모
- 스와이프 임계값: 가로 이동 40px 이상이며 세로 이동보다 클 때만 레벨 조작으로 인식(스크롤/탭과 구분)
- 들여쓰기 최대 깊이: 4단계
- 상세에서 편집한 내용은 뒤로가기 시 대시보드 카드 미리보기에 반영

## 7. 결정 사항 로그

1. 정적 SVG 목업(상세 → 대시보드) 2종으로 시각 방향 확정.
2. 클릭형 HTML 프로토타입으로 통합(대시보드 ↔ 상세 전환, 뷰 옵션 상단 우측 노출).
3. 카테고리 칩에서 "전체" 제거 → 카테고리 **단일·필수 선택**. 카드에서 카테고리 표기 제거.
4. 카드 제목 **한 줄 자동 맞춤**(축소 하한 11px, 이후 말줄임).
5. 상세 제목 **고정 높이 박스**(한 줄 큰 폰트 / 축소 두 줄).
6. 편집 모델 확정: **인라인 편집 + Enter 삽입**, **좌우 스와이프 들여쓰기**, 길게 누르기 메뉴.
